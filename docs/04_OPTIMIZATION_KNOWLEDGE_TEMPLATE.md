# TweakMind Optimization Knowledge Template

**Version:** 1.0

**Status:** LOCKED

> This document defines the standard structure for every optimization knowledge entry in TweakMind.

>

> Every optimization must follow this template.

>

> Fields may be expanded in the future.

>

> Existing fields should remain stable.

---

# 1. Identity

## Optimization Name

Official optimization name.

---

## Display Name

The name displayed to users.

Display Name changes according to the selected terminology mode.

The optimization itself never changes.

---

## Category

Primary category.

Example:

- Performance

- Gaming

- Security

- Privacy

- Compatibility

- Stability

---

## Technical Type

Examples:

- Windows Service

- Windows Feature

- Registry

- Group Policy

- Scheduled Task

- Power Configuration

- Driver Setting

---

## Supported Windows Versions

Specify supported versions.

Example:

- Windows 10

- Windows 11

- Windows 11 24H2

---

# 2. Understanding

## What is it?

Explain what this component is.

Maximum:

2 paragraphs.

---

## Why did Microsoft design it?

Explain Microsoft's design purpose.

Always explain the original intention before discussing optimization.

---

## How does it work?

Explain the mechanism using language appropriate for the selected terminology mode.

---

## Why do people optimize it?

Explain why users may want to change the default behavior.

Do not recommend.

Only explain.

---

# 3. Decision Information

## Benefits

Clearly explain what users gain.

Avoid vague wording.

Use measurable effects whenever possible.

---

## Trade-offs

Clearly explain what users lose.

Trade-offs are mandatory.

---

## Suitable For

List users who may benefit.

Examples:

- Gaming

- Low-end PCs

- Dedicated workstations

---

## Not Recommended For

List users who should probably keep the default configuration.

---

## Expected Value

Estimate practical value.

This value may be dynamically adjusted based on:

- hardware

- Windows version

- workload

- user goals

---

## Risk Level

Classify overall risk.

Explain why the risk exists.

---

## Recovery Difficulty

Describe how difficult recovery is.

---

## Restart Required

Specify whether restarting Windows is required.

---

# 4. Recovery

## Recovery Method

Describe how to restore the original behavior.

---

## Recovery Result

Explain what users should expect after recovery.

---

## Recovery Limitations

Explain whether recovery is immediate or delayed.

---

# 5. Evidence

## Official References

Microsoft documentation.

---

## Technical Analysis

Verified technical explanation.

---

## Community Validation

Widely confirmed community findings.

---

## Confidence Level

Overall confidence of the knowledge.

Example:

★★★★★

---

# 6. AI Assistance

## AI Summary

Short summary.

---

## AI Recommendation

Context-aware recommendation.

Recommendation should explain:

- why

- expected value

- possible downside

Never replace user choice.

---

## Related Optimizations

Examples:

- Windows Search

- SysMain

- Delivery Optimization

Explain why they are related.

# Completion Rule

An optimization knowledge entry is complete only when every section above has been completed.

Missing sections indicate incomplete knowledge.

Knowledge quality always takes priority over the number of supported optimizations.
